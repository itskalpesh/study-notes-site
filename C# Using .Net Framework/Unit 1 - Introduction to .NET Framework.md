# Introduction to .NET Framework

### .NET Framework 
The .NET Framework is a software development platform developed by Microsoft for building and running Windows applications. It provides libraries, runtime support, security, and language interoperability.

### Need of .NET Framework
- Simplifies application development.
- Supports multiple programming languages.
- Provides automatic memory management.
- Ensures code security.
- Offers a large class library.
- Enables rapid application development.

### Main Components of .NET Framework
The .NET Platform is built on three major components:

1. Common Language Runtime (CLR)
2. Common Type System (CTS)
3. Common Language Specification (CLS)

#### 1.Common Language Runtime (CLR)
The Common Language Runtime (CLR) is the execution engine of the .NET Framework. It manages the execution of .NET programs and provides services like memory management, security, exception handling, and garbage collection.

##### Functions of CLR
- Executes .NET programs.
- Converts Intermediate Language (IL) into machine code using Just-In-Time (JIT) Compiler.
- Performs automatic memory management.
- Provides Garbage Collection (GC).
- Handles exceptions.
- Provides security.
- Manages threads.
- Verifies code before execution.

#### 2. Common Type System (CTS)
The Common Type System (CTS) is a set of rules that defines how data types are declared, used, and managed in the .NET Framework. It ensures that all .NET languages use compatible data types.

Need of CTS
- Makes different .NET languages compatible.
- Ensures type safety.
- Enables language interoperability.
- Standardizes data types.

Types in CTS
1. Value Types
    - Store actual data.
    - Stored in stack memory.
    - Faster access.
    - Examples : int, char, float, double, bool

2. Reference Types
    - Store the address of an object.
    - Stored in heap memory.
    - Can hold null values.
    - Examples : class, string, array, object

#### 3. Common Language Specification (CLS)
The Common Language Specification (CLS) is a set of rules that every .NET language should follow so that code written in one language can be used by another .NET language.

Purpose of CLS
- Ensures language interoperability.
- Defines common programming rules.
- Allows different .NET languages to communicate.
- Improves code reusabilityLanguage-independent rules.
- Supports cross-language programming.
- Promotes reusable components.
- Ensures compatibility among .NET languages
